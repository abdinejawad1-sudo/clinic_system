# 🦷 موقع عيادة أسنان — نظام حجز مواعيد متكامل

موقع كامل لعيادة أسنان يتضمن:

- صفحة رئيسية تعرض معلومات العيادة، الخدمات، وطرق التواصل.
- نموذج حجز مواعيد للمرضى (الاسم، الهاتف، التاريخ، الوقت) مع منع تكرار حجز نفس الموعد.
- لوحة تحكم خاصة بالطبيب (محمية بكلمة مرور) لعرض وإدارة كل الحجوزات.
- إرسال **تذكير تلقائي عبر واتساب** للمريض قبل موعده (عبر Twilio WhatsApp API).
- قاعدة بيانات SQLite (ملف واحد، بدون الحاجة لتثبيت سيرفر قاعدة بيانات منفصل).

---

## 1. المتطلبات

- [Node.js](https://nodejs.org) نسخة 18 أو أحدث.
- حساب [Twilio](https://www.twilio.com/whatsapp) مجاني لتفعيل إرسال رسائل واتساب (اختياري لكن ضروري لتفعيل التذكير التلقائي).

## 2. خطوات التشغيل

```bash
# 1. تثبيت الاعتماديات
npm install

# 2. انسخ ملف الإعدادات وعدّله
cp .env.example .env
```

افتح ملف `.env` وعدّل القيم التالية على الأقل:

| المتغير | الوصف |
|---|---|
| `DOCTOR_USERNAME` / `DOCTOR_PASSWORD` | بيانات دخول الطبيب للوحة التحكم |
| `JWT_SECRET` | نص عشوائي طويل وسرّي (لا تشاركه مع أحد) |
| `TWILIO_ACCOUNT_SID` / `TWILIO_AUTH_TOKEN` | من لوحة تحكم Twilio |
| `TWILIO_WHATSAPP_FROM` | رقم واتساب المرسل من Twilio (في وضع Sandbox: `whatsapp:+14155238886`) |
| `REMINDER_HOURS_BEFORE` | كم ساعة قبل الموعد يُرسل التذكير (افتراضي 24) |

```bash
# 3. تشغيل السيرفر
npm start
```

افتح المتصفح على: `http://localhost:3000`

- الموقع العام: `http://localhost:3000`
- لوحة تحكم الطبيب: `http://localhost:3000/dashboard.html`

## 3. إعداد Twilio WhatsApp (للتذكير التلقائي)

1. أنشئ حساباً على [twilio.com](https://www.twilio.com).
2. من قسم **Messaging → Try it out → Send a WhatsApp message** فعّل الـ WhatsApp Sandbox.
3. اربط رقم واتساب تجريبي عبر إرسال كود التفعيل الظاهر في اللوحة.
4. انسخ `Account SID` و `Auth Token` إلى ملف `.env`.
5. عند الانتقال للإنتاج الفعلي، يجب طلب رقم واتساب معتمد (WhatsApp Business API) من Twilio بدل الـ Sandbox.

> ⚠️ ملاحظة: بدون إعداد Twilio بشكل صحيح، سيستمر الموقع بالعمل بشكل طبيعي (الحجز، منع التكرار، لوحة التحكم)، لكن مهمة التذكير التلقائي ستفشل بصمت وتُسجّل الخطأ في الطرفية (Terminal) فقط.

## 4. كيف يعمل منع تكرار الحجز؟

- قاعدة البيانات تحتوي قيد `UNIQUE(appt_date, appt_time)` يمنع تخزين حجزين لنفس التاريخ والوقت.
- الواجهة الأمامية أيضاً تجلب الأوقات المحجوزة (`/api/appointments/available`) وتعطّلها في القائمة قبل الإرسال، لتجربة استخدام أفضل.
- حتى لو وصل طلبان في نفس اللحظة تماماً، قاعدة البيانات ترفض الطلب الثاني وتُعيد رسالة خطأ واضحة.

## 5. كيف تعمل التذكيرات التلقائية؟

مهمة مجدولة (`node-cron`) تعمل كل 15 دقيقة (قابلة للتعديل عبر `REMINDER_CHECK_CRON`)، تفحص كل المواعيد المؤكدة التي:
- لم يُرسل لها تذكير بعد (`reminder_sent = 0`)
- ووقتها يقع خلال نافذة `REMINDER_HOURS_BEFORE` ساعة القادمة

ثم ترسل رسالة واتساب عبر Twilio وتُعلّم الموعد بأنه تم تذكيره حتى لا يتكرر الإرسال.

## 6. هيكل المشروع

```
dental-clinic/
├── server.js              # نقطة تشغيل السيرفر
├── db/

│   └── clinic.db          # ملف قاعدة البيانات (يُنشأ تلقائياً عند أول تشغيل)

│   ├── auth.js             # تسجيل دخول الطبيب
│   └── appointments.js     # إنشاء/عرض/تعديل/حذف المواعيد
├── middleware/
│   └── auth.js             # التحقق من صلاحية توكن الطبيب
├── services/
│   ├── whatsapp.js         # إرسال رسائل واتساب عبر Twilio
│   └── reminder.js         # المهمة المجدولة لإرسال التذكيرات
└── public/                 # الواجهة الأمامية (HTML/CSS/JS)
    ├── index.html          # الصفحة الرئيسية + نموذج الحجز
    ├── dashboard.html       # لوحة تحكم الطبيب
    └── css/js/...
```

## 7. النشر (Deployment)

يمكن نشر المشروع مباشرة على منصات مثل Render أو Railway أو أي VPS يدعم Node.js:
- تأكد من ضبط متغيرات البيئة (`.env`) على المنصة نفسها.
- ملف قاعدة البيانات `db/clinic.db` سيُنشأ تلقائياً — تأكد أن مساحة التخزين على المنصة **دائمة (persistent disk)** وليست مؤقتة، وإلا ستُفقد الحجوزات بعد كل إعادة نشر.

## 8. أمان مهم قبل النشر الفعلي

- غيّر `DOCTOR_PASSWORD` و `JWT_SECRET` لقيم قوية وفريدة.
- استخدم HTTPS دائماً في بيئة الإنتاج.
- إذا أردت أكثر من طبيب/مستخدم، يمكن تطوير جدول `users` بدل بيانات الدخول الثابتة الحالية في `.env`.
